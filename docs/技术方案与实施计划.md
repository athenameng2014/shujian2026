# 书间 V1 MVP — 技术方案与实施计划

## Context

书间是一款记录阅读轨迹的轻量级 PWA 工具。PRD 已完成，技术选型已确认：
- React + TypeScript + Vite
- Ant Design Mobile (UI)
- IndexedDB via localForage (本地存储)
- Google Books API + 本地上传兜底 (书籍封面)
- 无登录、纯本地、移动端优先 PWA

## 技术栈

| 层面 | 选型 | 说明 |
|------|------|------|
| 构建工具 | Vite 6 | 快速开发体验，PWA 插件支持好 |
| 前端框架 | React 19 + TypeScript | 用户选择 |
| UI 组件库 | Ant Design Mobile 5 | 移动端优化，开箱即用 |
| 样式方案 | CSS Modules + 自定义主题 | Antd Mobile 支持主题定制 |
| 路由 | React Router 7 | Tab 导航 + 子页面 |
| 本地存储 | localForage (IndexedDB) | 结构化数据 + 图片 Blob |
| 状态管理 | Zustand | 轻量，适合中小项目 |
| 封面获取 | Google Books API | 免费、无需鉴权 |
| PWA | vite-plugin-pwa | 离线支持、添加到主屏幕 |

## 数据模型

### Book（书籍）
```typescript
interface Book {
  id: string;            // uuid
  title: string;         // 书名
  author?: string;       // 作者
  coverUrl?: string;     // Google Books 封面 URL
  coverBlob?: Blob;      // 本地上传的封面（存 IndexedDB）
  color: string;         // 代表色，打卡日历用
  createdAt: number;     // 创建时间戳
}
```

### Log（打卡记录）
```typescript
interface Log {
  id: string;
  bookId: string;        // 关联书籍
  date: string;          // "2026-03-31"
  note?: string;         // 随想（选填）
  createdAt: number;
}
```

### Topic（专题）
```typescript
interface Topic {
  id: string;
  name: string;
  description?: string;
  bookIds: string[];     // 归属书籍 ID 列表
  createdAt: number;
}

// 书籍-专题关联状态存在 bookId 上，通过扩展或单独结构
interface TopicBook {
  topicId: string;
  bookId: string;
  status: 'want' | 'reading' | 'done';
  insight?: string;      // 核心观点/随想
}
```

## 实施步骤

### Step 1：项目脚手架
- 用 Vite 创建 React + TS 项目
- 安装依赖：antd-mobile, react-router, localforage, zustand, uuid
- 配置 PWA manifest 和 vite-plugin-pwa
- 搭建 Tab 导航骨架（日历/专题/我的三个 Tab）
- **产出**：`npm run dev` 能看到三个空白 Tab 页

### Step 2：数据层 + 封面服务
- 实现 localForage 初始化和 CRUD 函数
- 实现 Zustand stores
- 封装 Google Books API 搜索 + 封面获取
- 实现本地上传图片兜底
- **产出**：数据层可独立调用测试

### Step 3：Phase 1 核心流 — 打卡日历
- 打卡弹窗：选日期 → 搜书名/选已有书 → 完成（2步）
- 首次录入：书名输入 → Google Books 搜封面 → 选封面或本地上传
- 月历视图：格子填色（书籍代表色）、支持多书拼色
- 补打/编辑：点击历史日期可修改
- 今日 Timeline 显示
- **产出**：完整的打卡闭环

### Step 4：专题模块（Phase 2）
- 专题创建/编辑页面
- 书籍分配到专题 + 阅读状态管理
- 基础节点地图 UI（CSS 布局，书籍作为节点）
- 已读节点点亮 + 观点吸附
- **产出**：专题功能闭环

### Step 5：我的 + 数据展示（Phase 3）
- 书籍库总览页
- 阅读人格卡片：打卡频次、虚构/非虚构比例、专题偏好
- 截图分享（html2canvas）
- **产出**：V1 MVP 完整功能

## 验证方式
- `npm run dev` 启动后，手机浏览器访问 localhost（同局域网）
- 在手机上测试：添加书 → 打卡 → 查看日历 → 创建专题 → 分配书籍
- 验证 PWA：添加到主屏幕、离线打开
- 刷新页面后数据不丢失（IndexedDB 持久化）
