# 「书间」产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
**书间 (Between Books)**

### 1.2 产品定位
一款**记录阅读轨迹、发现阅读形态**的轻量级陪伴式工具。它不是传统的书单管理或沉重的笔记应用，而是通过最轻的交互，帮你把散乱的阅读行为具象化为清晰的“专题地图”和“打卡日历”，并在长期积累中看见自己的“阅读人格”。

### 1.3 适用平台
- **移动端优先的 Web 网页 (H5/PWA)**：无需下载 App，扫码或点击链接即可使用，体验上无限贴近原生应用（支持添加至手机主屏幕）。

---

## 2. 核心功能模块

### 2.1 模块一：阅读日历与打卡流 (Reading Calendar & Log)
**核心目标**：提供无压力的轻量化记录体验，生成极具视觉美感的月度数据。

**功能详情**：
1. **纯手打带图录入**：
   - 用户首次打卡一本书时，只需**纯手动输入书名**。
   - **系统自动联网获取**相应的书籍封面（可接入豆瓣或 Google Books API），减轻手动传图负担，同时保证界面的视觉统一与美观。
2. **灵活的日历打卡机制**：
   - 每日可记录“今天读了什么书”，支持单日**多条打卡**（例如今天同时读了虚构类与非虚构类各一本）。
   - **支持补打与编辑**：没有任何强迫症式的“断签惩罚”，用户可以随时补记过去的阅读状态，或修改已打卡的内容。
3. **可视化日历（月视图为主）**：
   - 以日历格子的形式呈现每月的阅读状况。
   - 视觉呈现：每本书支持自定义代表色，打卡时日历格子内填充相应书籍颜色的色块（如某天读了两本，格子内显示两种颜色的拼色或圆点）。一眼即可清晰掌握当月”读了多少天”、”哪几天读的是同一本书”。
4. **本月在读列表**：
   - 在日历视图下方，以列表形式展示当月正在阅读的所有书籍。
   - 按打卡频次降序排列（打卡最多的排在前面），每本书显示书名、封面缩略图（或代表色圆点）、当月打卡天数。


### 2.2 模块二：专题进度地图 (Thematic Progress Map)
**核心目标**：辅助“主题阅读”习惯，将读过的书和碎片认知串联成个人的知识网络。

**功能详情**：
1. **创建专题与书籍归属**：
   - 用户可自定义专题（如：“经济学入门”、“日本文学”、“思维模型”）。
   - 用户可将录入的书籍分配到一个或多个专题下（支持设置阅读状态：想读/在读/已读）。
2. **地图化视觉呈现**：
   - 摒弃枯燥的列表，采用“探索地图”或“知识星图”UI。
   - 书籍作为地图上的核心节点，当一本书被标记为“已读”，节点会被点亮。
   - 用户可在读完后记录一句“核心观点”或“随想”，这些内容会吸附在节点周围。随着专题内读完的书越来越多，系统（可结合AI）自动将相关书籍节点的观点相连，形成专题网络连接感。

### 2.3 模块三：365天阅读人格 (Reading Personality)
**核心目标**：通过持续的动作积累，给予用户具有情绪价值的正反馈。

**功能详情**：
1. **即时生成的阅读人格图谱**：
   - 基于用户的打卡频次、虚构与非虚构的比例、专题偏好等数据，随时（不局限于年底）可生成一份“阅读人格卡片”。
   - 包含内容：阅读连续性表现、类型偏好分布图、最具代表性的一本书/专题。
2. **分享属性**：
   - 卡片设计侧重审美与格调，方便用户一键生成图片分享至朋友圈或社交媒体，强调“你的阅读形塑了你”。

## 3. 信息架构层级 (Tab 导航)

系统底部为三大主 Tab 切换：

*   **[ 日历 ] (默认首页)**：展示月历视图、提供显著的【一键打卡】按钮、展示今日阅读Timeline；**展示本月在读列表**（当月所有在读书籍，按打卡频次排序，每本显示书名、封面/代表色、打卡天数）。
*   **[ 专题 ]**：以卡片或瀑布流形式展示当前进行中的阅读专题，点击进入具体的“节点地图”。
*   **[ 我的 ]**：个人基础设置、已读书籍库总览、书籍封面/颜色管理，以及【查看阅读人格图谱】入口。

---

## 4. 关键交互与 UI 调性要求

1. **极简录入，绝不强求**：打卡过程必须限制在 2 步以内（1.选日期 -> 2.选正在读的书 -> 完成）。写感悟是选填非核心项。
2. **Vibe 首位，审美驱动**：作为非工作向的小工具，界面风格需要“呼吸感”（大量留白、优雅的无衬线字体）。日历的填色要采用莫兰迪或高饱和度的柔和渐变色系，确保多书拼色时不杂乱。
3. **流畅的 Web 体验**：使用 React/Vue 或类似的现代前端框架实现 SPA (单页应用)，滑动顺滑，弹窗组件符合移动端手势直觉。

---

## 5. 开发里程碑建议 (V1.0 MVP 版本)

1. **Phase 1 (基础记录环)**：完成书名手打录入 + 封面自动获取API联调，跑通日历打卡、补签、修改功能。
2. **Phase 2 (专题整理环)**：完成专题创建与书籍分配逻辑，实现基础的“专题书籍节点地图”CSS样式渲染。
3. **Phase 3 (数据与展示)**：完成月历颜色渲染算法，梳理打卡数据逻辑并开发“阅读人格卡片”的 HTML2Canvas 截图分享功能。
