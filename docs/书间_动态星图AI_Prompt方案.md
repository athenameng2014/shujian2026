# 《书间》动态知识星图 AI Prompt 工程方案

本篇文档定义了《书间》App 专题模块两阶段 AI 生成的 **Prompt 详细设计** 及其**实机响应示例**。研发在落地时，只需替换 `输入变量`，即可要求大语言模型强制以规定好的 JSON 格式返回。

---

## 阶段一：新建专题时的图谱生成 (Phase 1 AI: Graph Generation)

**模型建议**：Gemini 1.5 Pro / GPT-4o 等具备较强发散与结构化输出能力的大模型。
**体温 (Temperature)**：建议设置为 `0.7`（需要一定想象力，但保持严谨）。

### 1.1 的 Prompt 核心构造

```text
角色设定：
你是一位具有极高审美、文笔优雅的知识架构师。你的任务是为一个阅读应用提取专题的“核心星图节点”。应用风格偏向莫兰迪色的简约与高级感，不枯燥，不呆板。

用户输入目标专题： 【{{专题名称}}】

任务要求：
1. 将该专题拆解为 3 到 4 个主要的「子领域」 (sub_disciplines)。
2. 为每个子领域，进一步抽象出 3 到 5 个最典型的「基石知识点」 (nodes)。节点名字应简练有力。
3. 为每个「基石知识点」撰写一句解释（description），字数控制在 40 字以内。语言必须优美、深刻、带有一点哲理与启发感，避免教科书式的生硬说教。

强制输出格式：
你必须且只能输出合法的 JSON 格式数据，不需要任何开头或结尾的寒暄（不需要代码块标记 ` ```json `，直接输出 ` { ` 开始)。JSON 结构必须严格如下：

{
  "topic": "{{专题名称}}",
  "sub_disciplines": [
    {
      "id": "sub_1",
      "name": "子领域名称",
      "nodes": [
        {
          "id": "node_1_1",
          "name": "知识点名称",
          "description": "优美的一句话解释。",
          "status": "unlit",
          "books_associated": []
        }
      ]
    }
  ]
}
```

## 阶段二：书籍锚定与点亮判定 (Phase 2 AI: Book Mapping)

**模型建议**：Gemini 1.5 Flash / GPT-4o-mini 等速度快、擅长文本提取及对比的模型。
**体温 (Temperature)**：建议设置为 `0.1`（追求精准的匹配与判定）。

### 2.1 的 Prompt 核心构造

```text
角色设定：
你是一个知识提炼工程师。你需要判断一本特定书籍的内核理念，是否覆盖了某个“知识星图”中的既定节点。

输入数据：
1. 待录入书籍标题：【{{书籍名}}】
2. 书籍摘要/梗概（可选，如无则交由模型本身背景知识）：【{{书籍简介}}】
3. 当前星图空置节点列表 JSON：【{{星图未点亮Nodes_JSON}}】

任务要求：
1. 分析书籍的核心主旨，提取它能显著解答、印证或探讨的理念。
2. 从“星图待点亮节点列表”中，找出哪些节点与该书籍主旨有着高度的相关性。
3. 门槛宁缺毋滥，如果相关性弱于 60%，请不要将节点算在内。一本书最多能点亮 4 个核心节点。

强制输出格式：
严格以合法的 JSON 数组输出被点亮节点的 `id`：

[
  "node_x_y",
  "node_z_w"
]
```

### 2.2 沙盒模拟输出：挂载一本【咖啡】新书

**输入情况假定：**
- `{{书籍名}}`：《蓝瓶咖啡的匠作》 
- `{{书籍简介}}`：讲述蓝瓶咖啡（Blue Bottle Coffee）如何从一家车库咖啡馆成为引领“第三波咖啡浪潮”的代表。书中详细拆解了每一杯手冲咖啡的冲煮细节，包括水流速率、金杯准则，并深刻讲述了品牌对单一产地（单一风土）微批次咖啡豆的极致追求。
- `{{星图未点亮Nodes_JSON}}`：传入上面刚才大模型生成的各个 Node 数组。

**AI 推断过程与输出：**
大模型比较书籍主旨后，精准锁定 `第三波浪潮`、`冲煮哲学相关` 以及 `探究产区风土` 的核心锚点。

输出结果：
```json
[
  "node_2_2", 
  "node_3_1",
  "node_3_2",
  "node_3_3"
]
```
*(注：对应的是“金杯准则”、“第三波浪潮”、“微批次”、“风土密码”)*

拿到这个数组后，系统后台就能把《蓝瓶咖啡的匠作》这一本书的信息挂载到 `books_associated` 里，并在前端触发这 4 颗星光的炫目“点亮”动效了。
